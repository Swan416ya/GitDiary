# Diary GitHub Sync - 产品设计文档

## 产品定位
一款回归本质的**数字日记本**，以传统纸质日记本的翻阅体验为核心，将数据安全地保存在用户自己的 GitHub 私有仓库中。

## 核心设计理念

### 1. 日记本隐喻 (Diary Metaphor)
- **日历视图**：像挂在墙上的月历，标记有日记的日期
- **翻阅体验**：左右滑动像翻页，带有轻微的纸张动画
- **那年今日**：打开旧日记时，发现去年今天的自己
- **封面质感**：启动页像一本真实的日记本封面

### 2. 极简主义
- 拒绝功能堆砌，聚焦"写日记"和"读日记"
- 无社交、无分享、无点赞
- 无广告、无内购提示
- 纯文本为主，支持简单的 Markdown 格式

### 3. 数据主权
- 用户完全拥有自己的数据
- Git 历史天然提供"时光机"功能
- 随时可以导出所有日记为 Markdown 文件

## 功能模块

### MVP 阶段 (Phase 1)

#### 1. 认证模块
- GitHub OAuth 登录
- 首次登录自动创建私有仓库（默认名：`my-diary`）
- Token 安全存储，支持退出登录清除数据

#### 2. 日记编辑
- 新建日记（默认当天日期）
- 纯文本编辑 + Markdown 预览
- 自动保存到本地
- 支持选择历史日期补写日记
- 每条日记包含：
  - 标题（可选）
  - 正文
  - 日期时间
  - 天气标记（可选，5种图标）
  - 心情标记（可选，5种表情）

#### 3. 日历浏览
- 月历视图，标记有日记的日期
- 点击日期查看/编辑当天日记
- 支持年份、月份切换
- 左右滑动切换月份（翻页感）

#### 4. 那年今日
- 首页展示历史上的今天
- 随机展示过往日记片段
- 支持查看完整条目

#### 5. 离线模式
- 无网络时正常编辑
- 本地 SQLite 缓存所有日记
- 有网络时自动同步
- 冲突解决：以最新时间戳为准

### Phase 2 (后续迭代)

#### 1. 日记列表
- 时间线视图（按时间倒序）
- 搜索功能（全文检索）
- 标签系统

#### 2. 数据统计
- 写作热力图（GitHub Contributions 风格）
- 心情统计图表
- 连续写日记天数

#### 3. 多媒体支持
- 照片插入（限制数量）
- 语音日记（可选）

#### 4. 增强体验
- 锁屏密码/指纹
- 本地数据导出
- 定期备份提醒

## 交互设计

### 1. 写日记流程
```
点击 + 按钮 → 进入编辑页（默认今天）
    ↓
选择日期（可选过去日期）
    ↓
标记天气/心情（可选）
    ↓
输入日记内容
    ↓
点击保存 → 本地存储 + 如在线则同步到 GitHub
    ↓
返回日历页，对应日期已标记
```

### 2. 浏览日记流程
```
月历页点击有标记的日期
    ↓
底部弹出当天日记预览
    ↓
点击查看完整内容 → 进入详情页
    ↓
左右滑动查看相邻日期的日记（翻页效果）
```

### 3. 那年今日
```
首页自动展示历史上的今天的日记
    ↓
左右滑动切换不同年份的同一天
    ↓
点击展开查看完整日记
```

## 视觉设计

### 色彩系统
采用 **Material You** 动态取色 + 日记本暖色调：

```
主色调：暖米色/奶油色 (Surface)
背景色：#FAF7F2 (纸张质感)
主色：深棕色 (Primary) #5D4037
次色：琥珀色 (Secondary) #FF8F00
强调色：赤陶色 (Tertiary) #E64A19
文字色：深灰/黑色 (On Surface)
```

### 字体
- **标题**：衬线字体（如思源宋体）增加书卷气
- **正文**：无衬线字体保证可读性
- **日记内容**：等宽字体模拟打字机效果（可选）

### 组件风格
- **卡片**：轻微阴影，圆角 12dp，模拟纸张
- **按钮**：填充式，圆角 24dp（药丸形）
- **输入框**：浅色背景，无外边框
- **图标**：线性图标，粗细一致

### 动画
- **翻页**：左右滑动切换月份时，轻微缩放 + 透明度变化
- **日记展开**：从日历日期位置展开到全屏（Hero Animation）
- **那年今日**：卡片轻微浮动，营造回忆感
- **保存成功**：纸张飞入文件柜的隐喻动画

### 单篇日记格式 (Markdown)
```markdown
---
date: 2024-01-01T20:30:00+08:00
weather: sunny
mood: happy
tags: ["新年", "计划"]
---

# 2024年的第一天

今天是新年的第一天，我做了一个重大的决定...

## 今日感悟

生活就像一本日记，每一页都值得认真对待。
```

## 用户体验原则

1. **零学习成本**：第一次打开就能写日记
2. **即时反馈**：每次操作都有视觉/触觉反馈
3. **容错设计**：误删可恢复，离线可编辑
4. **渐进式**：基础功能简单，高级功能不打扰
5. **情感化**：温暖的文案，贴心的提醒（如"你已经连续写了 7 天日记"）

## 竞品差异化

| 功能 | 普通日记 App | Diary GitHub Sync |
|------|-------------|-------------------|
| 数据存储 | 公司服务器 | 用户 GitHub 私有仓库 |
| 数据可携性 | 导出困难 | Git 仓库天然可导出 |
| 历史版本 | 通常无 | Git commit 自动保存 |
| 广告 | 有 | 无 |
| 成本 | 订阅制 | 免费（GitHub 免费额度） |
| 隐私 | 依赖平台承诺 | 物理上用户完全掌控 |
